# Agent Instructions — arquivo-mcp

## Critical Constraints

- **MCP SDK version** pinned to `>=1.0.0 <1.28.0` in package.json. Newer versions (>=1.28) cause JSON-RPC error -32602. Do not upgrade without testing.
- **Error responses** must include `type: 'text'` in every content object. Missing this causes -32602 validation failures.
- **Rate limiting**: Server throttles to 1 req/sec by default. Configure via `MAX_REQUESTS_PER_SECOND` env var.
- **Two client setups**:
  - **Claude Desktop**: Requires global `npm install -g` and config merge via `make install-claude`.
  - **OpenCode**: Requires local `opencode.json` (generated by `make install-opencode`) pointing to `dist/index.js`.

## Development Workflow

```bash
make setup      # npm ci
make build      # TypeScript → dist/
make test       # Vitest (unit + integration if set)
make lint       # ESLint
make format     # Prettier
make check      # Full gate: docs-check + code-check + lint-check + test-check
```

**Pre-commit**: run all four targets in order. `make check` fails if any required doc (`docs/VISION.md`, `docs/PERSONAS.md`, `docs/REQUIREMENTS.md`, `docs/ROADMAP.md`) is missing or appears empty.

## Project Structure

- `src/index.ts` — stdio MCP server entry (Claude/Cursor)
- `src/server/http-server.ts` — HTTP transport entry (optional)
- `src/tools/*.ts` — tool implementations; each returns `{ content: Array<{ type: 'text'; text: string }> }`
- `src/client/ArquivoClient.ts` — API client with retry, throttling, timeout
- `dist/` — compiled output; must be rebuildable at any time

## Tool Implementation Rules

1. Every tool function must return `{ content: [...] }` where each item has `type: 'text'` and `text: string`.
2. Use `truncateToTokens(output, 8000)` (from `src/utils/tokens.ts`) to respect token budget.
3. Add unit test in `tests/unit/` and integration test in `tests/integration/` for any API change.
4. Update `inputSchema` in `src/index.ts` AND `src/server/http-server.ts` if adding/removing parameters.

## Gotchas

- `arquivo_get_page_content` often returns Wayback wrapper JS instead of original page content because actual content is in an iframe. This is a known limitation; extracted text link selectors are best-effort.
- Intermittent -32602 errors with OpenCode typically mean wrong MCP SDK version. Ensure `package.json` constraint is respected.
- Integration tests hit real Arquivo.pt API; may fail on rate limits or API downtime. Set `INTEGRATION=false` to skip (default).
- The Makefile's `make check` requires the four docs to exist and contain keywords (`Problema` in VISION.md, `Persona` in PERSONAS.md, etc.). Don't remove these files.

## Setup for OpenCode

1. `make build`
2. `make install-opencode` (interactive; creates `opencode.json` in current dir)
3. Run `opencode` in that directory.

The generated `opencode.json` points to `dist/index.js`. If you move the project, regenerate or update the absolute path.

## Environment Variables

- `MAX_REQUESTS_PER_SECOND` — outgoing API rate (default: 1)
- `LOG_LEVEL` — `debug` | `info` | `warn` | `error` (default: `info`)
- `HTTP_PORT` — for `--http` mode (default: 3000)

## When Modifying

- If changing tool parameters → update both server files' `inputSchema`.
- If changing output format → ensure `type: 'text'` present everywhere.
- If adding new tool → add to `src/tools/`, export in `src/tools/index.ts`, register in both servers.
- If bumping MCP SDK → test thoroughly; <1.28.0 is required for current validation handling.

## Troubleshooting

- Tools not appearing → restart client, verify config file location, check server process is running.
- Build fails → Node.js 18+ required; delete `node_modules` and `dist`, run `make setup && make build`.
- Lint fails → run `make format` to auto-fix Prettier issues; resolve remaining ESLint errors manually.
- `make check` fails on docs → ensure `docs/VISION.md` etc. have substantive content (not placeholders).

## Important Notes

- Repository uses ES modules (`"type": "module"`). Keep `.ts` files as ESM.
- Tests use Vitest with 30s timeout.
- The server is single-threaded; long-running `get_page_content` may block but token truncation limits exposure.
- No database; all state is in-memory. Safe to restart frequently.

## Client Configuration

The `ArquivoClient` defaults (configurable via constructor options):

| Option                 | Default                              | Description                                                                                                     |
| ---------------------- | ------------------------------------ | --------------------------------------------------------------------------------------------------------------- |
| `maxRequestsPerSecond` | 1 (or `MAX_REQUESTS_PER_SECOND` env) | Rate limiting; 1 token/sec refill                                                                               |
| `maxRetries`           | 4                                    | Number of retry attempts on retryable errors                                                                    |
| `timeoutMs`            | 120000                               | Base timeout (120s). **Adaptive**: actual timeout = max(baseTimeout, 30s + (maxItems × 600ms)), clamped 30–180s |
| `keepalive`            | true (enabled)                       | HTTP keep-alive for connection reuse (when supported)                                                           |

Retry policy: exponential backoff with 2s base, 0.8–1.2 jitter. Respects `Retry-After` on 429.

Retryable errors: HTTP 429/5xx, network errors (ECONNABORTED, ENETUNREACH), AbortError.
